# プロジェクトの定義

## 現在の段階

Akari CEは独立したソフトウェアプロジェクトとして準備を進めています。今は個別機能の実装よりも、プロジェクトの目的と境界を丁寧に定めることを優先します。

## 次の作業

1. 解決したい課題と利用者を整理する。
2. 対象範囲と対象外を明確にする。
3. 構成要素と責任の境界を決める。
4. 他のDokoniプロジェクトとの接点を文書化する。
5. アーキテクチャ、開発方針、最初のロードマップを決める。

## 文書化の方針

確認できた公開情報はこのリポジトリで管理します。一方、検討途中の作業知識によって将来の設計を意図せず固定しないようにします。
